## 使用说明

本文档为**通用执行框架**，每次测试需基于此生成具体执行方案（`联调测试.{mmdd}.docker.{num}.md`）。

**模板引用规则**：
- 结构化日志 → 参照模板§5.1+§5.2（4种核心事件）

- 报告输出 → 参照模板§9.1~§9.7（7节结构）

## 通用执行框架

### Phase 1: 部署验证

**目标**：确保所有组件 running + healthy 且配置冻结

**步骤**：
1. 创建/验证 Docker 网络隔离
2. Server rebuild + 启动
3. Agent 镜像 rebuild
4. 清理旧 data（避免状态污染）
5. 启动全部 agent 容器
6. 验证所有容器 healthy

**验证标准**：
- `docker ps` 显示全部容器 running
- `GET /health` 返回 healthy
- 无 panic/error 日志

---

### Phase 2: 角色创建

**目标**：创建具有完整背景的新角色，验证注册流程

**步骤**：
1. 对每个 agent 依次调用：
   - `POST /api/v1/character/generate` — LLM 生成角色
   - `POST /api/v1/character/register` — 注册到 server
2. 验证注册成功（`GET /api/v1/character` 返回 agent_id）

**验证标准**：
- 每个 agent 均成功创建角色
- 角色具有完整背景（非残缺/旧角色复用）

---

### Phase 3: 监控与指标采集

**监控周期**：每 `X` min 执行一轮（建议 5-30min，可根据测试时长调整）

**3a. 健康状态检查**
- 每个 agent: `GET /api/v1/character` — 角色状态
- 每个 agent: `GET /api/v1/state` — WorldState 快照
- Server 全局视图（如有）
- Docker logs — 近 N 分钟日志

**3b. 事件链路记录（模板§5.1+§5.2）**
按4种核心事件类型记录：
- Action：决策上下文 + 提交的操作
- ToolResult：ok/timeout/error + latency
- StateDiff：action 前后状态差分
- Error：异常分类

**3c. 死亡处理**
- 触发条件：收到 `AgentDied` 事件
- 处理流程：
  1. 调用 `POST /api/v1/character/rebirth`
  2. 等待 reconnect
  3. 生成新角色 + 注册
  4. 记录死亡原因 + trace_id

**3d. 恶性 Bug 判定**
满足任一立即中断：
- 关键流程无法推进（无法创建角色/action 死循环）
- 数据损坏或状态不可恢复
- 大面积超时/错误导致样本不可用

**处理流程**：中断 → 记录复现步骤 + trace_id → 修复 → 回归验证 → 继续

---

### Phase 4: 报告输出

**输出文件**：`logs/测试报告/联调测试.{mmdd}.docker.{num}.md`

**节数**：9.1 ~ 9.7（按模板§9）

| 报告节 | 内容 |
|--------|------|
| §9.1 验收参数区 | 冻结的阈值 + 部署信息 |

| §9.7 问题清单 | P0/P1/P2 + trace_id + 修复状态 |

## 核心假设验收标准速查

### H1 自主决策（模板§6.1）
| 指标 | 公式 |
|------|------|
| DecisionCoverage | 完成 Decision 次数 / 触发决策点次数 |
| ActionSuccessRate | tool_status=ok 次数 / 提交 Action 次数 |
| AttributionRate | 有 StateDiff 且可归因次数 / tool_status=ok 次数 |

**决策点**：事后从 Action 链路反推分类（模板已改预定义为事后观测）

### H2 行为涌现（模板§6.2）
- 只接受系统级指标：TaskCompletionRate / ParallelWorkRatio / OscillationRate 等
- 必须 Baseline vs Intervention A/B 实验
- 禁止仅凭文本叙事判定

### H3 技术可行性（模板§6.3）
- 延迟 TP50/TP95、错误率/超时率、成本/Tokens 分位数
- SLO 上限需预先冻结
