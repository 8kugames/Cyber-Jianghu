# Agent 认知架构

## ActorSoul + ReflectorSoul 双 Soul 设计

Agent 内部采用**双 Soul 架构**，模拟人类心理学中的本我和超我：

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent (单进程)                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   shared ReviewStore (内存共享)                              │
│         │                                                    │
│         │ (内存共享)                                          │
│         ▼                                                    │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  ActorSoul (行动之魂/本我) │ ReflectorSoul (反思之魂/超我) │   │
│   │  ─────────────────────────────────────────────────  │   │
│   │  • 决策：decide()         │ • 审查：poll ReviewStore    │   │
│   │  • 提交：submit_for_review() │ • LLM 判断：approve/reject  │   │
│   │  • 等待：await approval    │ • 反馈：submit_review()     │   │
│   │  • 发送：send_intent()     │ • 超时：timeout → auto-approve │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### ActorSoul (行动之魂)

- 职责：生成意图，执行行动
- 输入：WorldState + MemoryContext
- 输出：Intent
- 特点：追求即时满足，类似弗洛伊德的"本我"

### ReflectorSoul (反思之魂)

- 职责：审查意图，道德判断
- 输入：PendingReview (从 ReviewStore)
- 输出：ReviewDecision (Approved/Rejected)
- 特点：代表长期利益和价值观，类似弗洛伊德的"超我"

### ReviewStore (共享记忆)

> 模块路径：`crates/agent/src/soul/reflector/store.rs`

```rust
pub struct ReviewStore {
    pending: HashMap<Uuid, PendingReviewEntry>,  // 待审查队列
    results: HashMap<Uuid, ReviewResult>,         // 审查结果
    config: ReviewConfig,                          // 超时配置
}
```

**审查流程**：
1. ActorSoul 生成 Intent 后调用 `submit_for_review()`
2. Intent 被添加到 `pending` 队列
3. ReflectorSoul 轮询 `pending` 队列
4. ReflectorSoul 使用 LLM 审查后调用 `submit_review()`
5. ActorSoul 收到审查结果（Approved/Rejected/TimeoutApproved）

### 超时机制

- 默认超时：30 秒
- 超时后自动批准（`TimeoutApproved`）
- 防止 ActorSoul 阻塞太久导致 Tick 过期

## 三层记忆系统

```
语义记忆 (Semantic)
  - 向量嵌入存储
  - 支持语义相似度检索
  - 跨事件知识聚合
        ▲
        | 遗忘/归档
        |
情景记忆 (Episodic)
  - 事件序列记录
  - 时间顺序组织
  - 经验累积
        ▲
        | 整合
        |
工作记忆 (Working)
  - 当前上下文
  - 活跃思考
  - 短期信息保持
```

### 各层说明

**工作记忆 (Working Memory)**
- 最近 N 条事件（默认 20）
- 内存环形缓冲
- 快速读写

**情景记忆 (Episodic Memory)**
- 重要事件
- SQLite 持久化
- 艾宾浩斯遗忘曲线

**语义记忆 (Semantic Memory)**
- 向量嵌入存储
- 语义相似度检索
- 跨事件知识聚合

**归档记忆 (Archive Memory)**
- 被遗忘的记忆
- 可通过努力召回

### 艾宾浩斯遗忘

每 84 tick 执行一次，公式：`R(t) = e^(-t/S)`

## 意图验证

1. **规则引擎验证** - 基于人格特质和当前状态
2. **认知验证** - 基于 LLM 的合理性判断
3. **ReflectorSoul 审查** - 基于道德和世界观的二次判断

**意图验证 vs ReflectorSoul 审查的区别**：

| 特性 | 意图验证 (Validator) | ReflectorSoul 审查 |
|------|----------------------|-------------------|
| 触发时机 | 决策前，重试机制 | 决策后，一次性 |
| 失败处理 | 重试（最多 5 次） | 拒绝并返回 idle |
| 超时行为 | 强制 idle | 自动批准 |
| LLM 调用 | 是 | 是 |
| 关注点 | 规则符合性 | 道德合理性 |

验证结果：
- Approved：验证通过，包含 reason 和 narrative
- Rejected：验证拒绝，包含 reason 和 rejection_type

验证循环：最多 5 次重试，连续 3 次拒绝后强制 idle
