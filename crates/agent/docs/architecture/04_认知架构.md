# Agent 认知架构

## 三魂架构（人魂 + 天魂）

> **架构统一**: Cognitive 与 Claw 模式共享相同的三魂架构和认知引擎，仅 LLM 客户端实现不同。初始化流程统一为 Phase 1（LLM 客户端创建）+ Phase 2（共享初始化：CognitiveEngine、OutcomeMemory、ChaosGenerator、回调注册）。

Agent 内部采用**三魂架构**，人魂直连 WorldState 输出结构化 Intent：

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent (单进程)                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   lifecycle 主循环（同步串联）                               │
│         │                                                    │
│         ▼                                                    │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  人魂 ActorSoul          │  天魂 ReflectorSoul       │   │
│   │  ───────────────────────│────────────────────────── │   │
│   │  直连 WorldState         │  分级审查                  │   │
│   │  输出结构化 Intent       │  action_type 合法性        │   │
│   │  action_type=eat         │  → RuleEngine              │   │
│   │  item_id=mantou          │  → LLM 人设/世界观         │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                              │
│   驳回反馈叙事化：技术性错误 → 自然语言 → 人魂重试           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 人魂 ActorSoul (行动之魂)

- 职责：直连 WorldState，生成结构化 Intent
- 输入：`WorldState`（精确数据：item_id, node_id, entity UUID）+ `memory_context` + 叙事化反馈
- 输出：`action_type=eat`, `action_data={"item_id": "mantou"}`
- 特点：直接看到精确 ID，无需中间翻译步骤

### 天魂 ReflectorSoul (守护之魂)

- 职责：分级审查格式化 Intent
- 输入：人魂输出的结构化 Intent
- 输出：Approved / Rejected(reason)
- **分级审核策略**：
  1. **Skip**: 仅 Layer1+2 确定性校验（action_type 合法性 + RuleEngine）
  2. **Always**: 完整三层审核（Skip + LLM 人设/世界观）
  3. **Adaptive**: 动态判断是否需要 LLM（限制区域/高价值物品检测）
- **决策字段映射**（数据驱动）:
  - `target_location` → 检查 `restricted_area_keywords`
  - `item_id` → 检查 `high_value_item_keywords`
  - 其他字段 → 默认需要 LLM 审核

### 地魂 (能力之魂)

- 职责：提供 tool calling 工具池，行动落地层
- 位置：嵌入 ActorSoul，人魂通过 tool calling 调用行动能力
- 特点：人魂输出结构化 Intent 时，地魂的 tool calling 工具提供行动的落地执行能力

### 驳回反馈增强

RuleEngine 规则失败时，自动从 context 提取可用 items/nodes 附在错误消息中（数据驱动，模板来自 `prompt_templates.yaml`）：

| 技术性原因 | 增强反馈（附上下文） |
|-----------|-----------|
| 吃东西失败：物品ID无效 | 吃东西失败：物品ID无效。背包可用物品：[馒头, 水]。请使用 eat 命令，item_id 必须是上述之一。 |
| 移动失败：目标地点ID无效 | 移动失败：目标地点ID无效。可前往的地点：[龙门客栈, 后院, 厨房]。请使用 move 命令，target_location 必须是上述之一。 |

### 审查流程

```
ActorSoul → 结构化 Intent (含精确 ID)
    → ReflectorSoul.validate_with_reflector()
        ├── Layer 1: validate_action_type()
        ├── Layer 2: RuleEngine — item_id/node_id 有效性
        └── Layer 3: LLM — 人设/世界观（分级策略：Skip/Always/Adaptive）
    → Approved: send_intent()
    → Rejected: narrativize_rejection() → ActorSoul 重试（最多 3 次）
```

### Prompt 模板系统

人魂的 prompt 从 `prompt_templates.yaml` 加载，数据驱动替代硬编码。`{variable}` 占位符替换，`required_sections` 启动时 fail-fast 校验。

```yaml
templates:
  actor_direct:
    required_sections: [header, world_state, task, survival_rules, ...]
    truncation:
      planning_description: 100
      content_hint: 30
      summary_window: 50
```

### 生存压力注入

hunger/thirst 低于阈值时，lifecycle.rs 向 memory_context 注入紧急信号模板（口渴优先）：

```
【紧急】你极度饥饿（当前 15/100），必须立即进食！背包可用物品：馒头、水。
【紧急】你极度口渴（当前 10/100），附近无水源！移动到有水的地点或采集。
```

### 理智值注入

sanity 低于阈值时注入精神状态描述，引导 LLM 产生混乱行为（不绕过三魂架构）：

```
【心神不宁】你的精神状态有些恍惚（理智 45/100），思绪开始混乱。（sanity ≤ 50）
【精神崩溃】你已失去理智控制（理智 10/100）！行为可能变得疯狂。（sanity ≤ 30）
```

### multi-Intent Pipeline

单 tick 可提交多 Intent，顺序执行，失败回滚：

```
Tick 广播
    │
    ▼
ActorSoul 生成结构化 Intents（多个，含精确 ID）
    │
    ▼
ReflectorSoul 分级审核
    │
    ├── Skip: validate_rules_only() — 仅 Layer1+2
    ├── Always: 完整三层审核
    └── Adaptive: restricted_area/high_value 检测 → 决策是否需要 LLM
    │
    ▼
IntentBatch 提交到 Server
    │
    ▼
Server 执行 Pipeline，失败时 Sagas 回滚
```

**IntentBatchConfig 配置**：
- `max_intents_per_tick`: 每 tick 最大 Intent 数（默认 5）
- `max_retries`: 三魂循环最大重试次数（默认 3）
- `pipeline_execution_enabled`: 是否启用 Pipeline 执行
- `partial_execution_enabled`: 是否允许部分执行

## 三层记忆系统

```
语义记忆 (Semantic)
  - 向量嵌入存储
  - 支持语义相似度检索
  - 跨事件知识聚合
        ▲
        | 遗忘/归档
        |
情景记忆 (Episodic)
  - 事件序列记录
  - 时间顺序组织
  - 经验累积
        ▲
        | 整合
        |
工作记忆 (Working)
  - 当前上下文
  - 活跃思考
  - 短期信息保持
```

### 各层说明

**工作记忆 (Working Memory)**
- 最近 N 条事件（默认 20）
- 内存环形缓冲
- 快速读写

**情景记忆 (Episodic Memory)**
- 重要事件
- SQLite 持久化
- 艾宾浩斯遗忘曲线

**语义记忆 (Semantic Memory)**
- 向量嵌入存储
- 语义相似度检索
- 跨事件知识聚合

**归档记忆 (Archive Memory)**
- 被遗忘的记忆
- 可通过努力召回

### 艾宾浩斯遗忘

每 84 tick 执行一次，公式：`R(t) = e^(-t/S)`

## LLM 容错与降级

ActorSoul 的 LLM 调用通过 `FallbackLlmClient` 实现 3 层降级：

```
主模型 → fallback_models 轮转 → idle intent（武侠化 thought_log）→ tick 超时强制 idle
```

**FallbackLlmClient** (`component/llm/client.rs`):
- 包装多个 `LlmClient`，主模型失败时自动切换备用模型
- 所有模型共享同一 `provider`/`api_key`，仅 `model` 不同
- Sticky fallback：成功后锁定该模型直到它也失败
- 触发条件（`should_fallback`）：403（额度耗尽）、429（限速）、`AllocationQuota`、连接/请求失败
- `enable_thinking` disable list：kimi、qwen、qwq、qvq 系列模型要求非流式调用禁用 thinking

**Idle 降级**：
- 所有 LLM 模型均失败 → ActorSoul 返回 idle intent
- Tick deadline 超时（决策耗时超过 tick 剩余时间）→ 强制 idle
- thought_log 使用武侠化描述，不暴露技术细节

**配置**（`agent.yaml`）：
```yaml
llm:
  provider: dashscope
  model: qwen-max
  fallback_models:        # 可选，同 provider/api_key
    - qwen-plus
    - qwen-turbo
```
