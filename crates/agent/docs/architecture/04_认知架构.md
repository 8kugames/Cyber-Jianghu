# Agent 认知架构

## 三魂架构（人魂 + 天魂 + 地魂）

Agent 内部采用**三魂架构**，实现叙事与结构化的分离：

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent (单进程)                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   lifecycle 主循环（同步串联）                               │
│         │                                                    │
│         ▼                                                    │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  人魂 ActorSoul │ 天魂 IntentTranslator │ 地魂 ReflectorSoul │   │
│   │  ─────────────────────────────────────────────────  │   │
│   │  叙事意图生成     │ 叙事→格式化翻译      │ 分级审查            │   │
│   │  纯推理，无 ID   │ 精确 ID 映射         │ action_type         │   │
│   │  action_type=    │ action_type=eat     │ → RuleEngine        │   │
│   │    "narrative"   │ item_id=mantou      │ → LLM 人设/世界观    │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                              │
│   驳回反馈叙事化：技术性错误 → 自然语言 → 人魂重试           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 人魂 ActorSoul (行动之魂)

- 职责：生成纯叙事意图
- 输入：WorldState（无技术 ID）+ MemoryContext + 叙事化反馈
- 输出：`action_type="narrative"`, `action_data={"narrative": "吃馒头充饥"}`
- 特点：完全不接触 item_id/node_id 等技术信息

### 天魂 IntentTranslator

- 职责：LLM 翻译叙事为格式化 Intent
- 输入：叙事文本 + WorldState（含精确 ID）
- 输出：`action_type=eat`, `action_data={"item_id": "mantou"}`
- 特点：连接叙事世界和游戏系统的桥梁
- 内置 30 秒超时保护

### 地魂 ReflectorSoul (反思之魂)

- 职责：分级审查格式化 Intent + NarrativeGenerator 叙事生成
- 输入：翻译后的 Intent
- 输出：Approved / Rejected(reason)
- **分级审核策略**：
  1. **Skip**: 仅 Layer1+2 确定性校验（action_type 合法性 + RuleEngine）
  2. **Always**: 完整三层审核（Skip + LLM 人设/世界观）
  3. **Adaptive**: 动态判断是否需要 LLM（限制区域/高价值物品检测）
- **NarrativeGenerator**: LLM 生成叙事上下文，语义缓存，泄露检测
- **决策字段映射**（数据驱动）:
  - `target_location` → 检查 `restricted_area_keywords`
  - `item_id` → 检查 `high_value_item_keywords`
  - 其他字段 → 默认需要 LLM 审核

### 驳回反馈叙事化

技术性驳回原因在传回人魂前被转换为自然语言：

| 技术性原因 | 叙事化反馈 |
|-----------|-----------|
| 吃东西失败：物品ID无效 | 你想吃喝点东西，但发现手边没有合适的物品 |
| 移动失败：目标地点ID无效 | 你想要移动到别处，但发现那条路走不通 |
| action 不在合法列表 | 你想做一件事，但似乎无法如愿 |

人魂永远不看到 "item_id"、"node_id" 等技术性信息。

### 审查流程

```
ActorSoul → narrative Intent
    → IntentTranslator.translate() → 格式化 Intent
    → ReflectorSoul.validate_with_reflector()
        ├── Layer 1: validate_action_type() — "narrative" sentinel 被拦截
        ├── Layer 2: RuleEngine — item_id/node_id 有效性
        └── Layer 3: LLM — 人设/世界观（分级策略：Skip/Always/Adaptive）
    → Approved: send_intent()
    → Rejected: narrativize_rejection() → ActorSoul 重试（最多 3 次）
```

### multi-Intent Pipeline

单 tick 可提交多 Intent，顺序执行，失败回滚：

```
Tick 广播
    │
    ▼
ActorSoul 生成 Narrative Intents（多个）
    │
    ▼
IntentTranslator 翻译为格式化 Intents
    │
    ▼
ReflectorSoul 分级审核
    │
    ├── Skip: validate_rules_only() — 仅 Layer1+2
    ├── Always: 完整三层审核
    └── Adaptive: restricted_area/high_value 检测 → 决策是否需要 LLM
    │
    ▼
IntentBatch 提交到 Server
    │
    ▼
Server 执行 Pipeline，失败时 Sagas 回滚
```

**IntentBatchConfig 配置**：
- `max_intents_per_tick`: 每 tick 最大 Intent 数（默认 5）
- `max_retries`: 三魂循环最大重试次数（默认 3）
- `pipeline_execution_enabled`: 是否启用 Pipeline 执行
- `partial_execution_enabled`: 是否允许部分执行

## 三层记忆系统

```
语义记忆 (Semantic)
  - 向量嵌入存储
  - 支持语义相似度检索
  - 跨事件知识聚合
        ▲
        | 遗忘/归档
        |
情景记忆 (Episodic)
  - 事件序列记录
  - 时间顺序组织
  - 经验累积
        ▲
        | 整合
        |
工作记忆 (Working)
  - 当前上下文
  - 活跃思考
  - 短期信息保持
```

### 各层说明

**工作记忆 (Working Memory)**
- 最近 N 条事件（默认 20）
- 内存环形缓冲
- 快速读写

**情景记忆 (Episodic Memory)**
- 重要事件
- SQLite 持久化
- 艾宾浩斯遗忘曲线

**语义记忆 (Semantic Memory)**
- 向量嵌入存储
- 语义相似度检索
- 跨事件知识聚合

**归档记忆 (Archive Memory)**
- 被遗忘的记忆
- 可通过努力召回

### 艾宾浩斯遗忘

每 84 tick 执行一次，公式：`R(t) = e^(-t/S)`

## LLM 容错与降级

ActorSoul 的 LLM 调用通过 `FallbackLlmClient` 实现 3 层降级：

```
主模型 → fallback_models 轮转 → idle intent（武侠化 thought_log）→ tick 超时强制 idle
```

**FallbackLlmClient** (`component/llm/client.rs`):
- 包装多个 `LlmClient`，主模型失败时自动切换备用模型
- 所有模型共享同一 `provider`/`api_key`，仅 `model` 不同
- Sticky fallback：成功后锁定该模型直到它也失败
- 触发条件（`should_fallback`）：403（额度耗尽）、429（限速）、`AllocationQuota`、连接/请求失败
- `enable_thinking` disable list：kimi、qwen、qwq、qvq 系列模型要求非流式调用禁用 thinking

**Idle 降级**：
- 所有 LLM 模型均失败 → ActorSoul 返回 idle intent
- Tick deadline 超时（决策耗时超过 tick 剩余时间）→ 强制 idle
- thought_log 使用武侠化描述，不暴露技术细节

**配置**（`agent.yaml`）：
```yaml
llm:
  provider: dashscope
  model: qwen-max
  fallback_models:        # 可选，同 provider/api_key
    - qwen-plus
    - qwen-turbo
```
