# Agent 通信协议

## 概述

Agent 提供两种通信通道：
- **WebSocket（主通道）**：OpenClaw **必须**通过 WebSocket 连接 Agent，确保 Tick 实时同步
- **HTTP API（辅助功能）**：用于数据查询、Web 面板、调试等

## Agent → Server (WebSocket 客户端)

Agent 内部集成 WebSocket 客户端，连接到 Server：
```
ws://cyber-jianghu-server:23333/ws?device_id={uuid}&token={auth_token}
```

## OpenClaw → Agent (WebSocket Server)

Agent 提供 WebSocket 服务端，OpenClaw **必须**连接：
```
ws://localhost:23340/ws
```

### 下行消息 (DownstreamMessage)

```json
// Tick 通知
{
  "Tick": {
    "tick_id": 123,
    "deadline_ms": 1710937800000,
    "state": { ... },
    "context": "叙事化上下文..."
  }
}

// Tick 截止通知
{
  "TickClosed": {
    "tick_id": 123,
    "received": true
  }
}
```

### 上行消息 (UpstreamMessage)

```json
// 提交意图
{
  "Intent": {
    "tick_id": 123,
    "action_type": "idle",
    "action_data": {},
    "priority": 1
  }
}
```

## OpenClaw → Agent (HTTP API - 辅助功能)

> ⚠️ **重要警告**: HTTP API **仅用于调试**，OpenClaw **禁止**使用 HTTP API 提交意图。
>
> **原因**：
> 1. HTTP 轮询存在延迟，无法保证 tick_id 实时性
> 2. Server 只接受**当前 tick** 的意图（硬性要求）
> 3. HTTP 请求与 WebSocket 推送存在时序竞争
>
> **必须使用 WebSocket 提交意图**，见上文 "上行消息 (UpstreamMessage)"

### Intent 提交（仅用于调试）

```
POST /api/v1/intent
Content-Type: application/json

{
  "tick_id": 123,
  "action_type": "idle",
  "action_data": {},
  "priority": 1
}

Response:
{
  "status": "submitted",
  "intent_id": "uuid",
  "tick_id": 123,
  "action_type": "idle"
}
```

注意：此 API 只确认收到意图，不返回执行结果。执行结果通过 `GET /api/v1/state` 的 `events_log` 获取。

## 决策模式

### WebSocket 模式（硬性要求）

OpenClaw **必须**通过 WebSocket 连接 Agent，确保 Tick 实时同步：

1. Agent 连接到 Server WebSocket（内部集成，默认启用）
2. OpenClaw 连接到 Agent WebSocket Server (`ws://localhost:23340/ws`)
3. Agent 接收 Server 的 Tick 消息并转发到 OpenClaw
4. OpenClaw 实时响应并提交意图
5. Agent 将意图转发到 Server

### HTTP API（辅助功能）

HTTP API 用于数据查询和辅助功能，**不是** WebSocket 的替代方案：

- `GET /api/v1/state` - 查询当前世界状态
- `GET /api/v1/context` - 获取叙事化上下文
- `GET /api/v1/memory/*` - 记忆管理
- `GET /api/v1/relationship/*` - 关系管理
- Web 面板、调试等
