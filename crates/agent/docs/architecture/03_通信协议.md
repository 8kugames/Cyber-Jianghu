# Agent 通信协议

## OpenClaw → Agent (HTTP API)

### Intent 提交

```
POST /api/v1/intent
Content-Type: application/json

{
  "tick_id": 123,
  "action_type": "idle",
  "action_data": {},
  "priority": 1
}

Response:
{
  "status": "submitted",
  "intent_id": "uuid",
  "tick_id": 123,
  "action_type": "idle"
}
```

注意：此 API 只确认收到意图，不返回执行结果。执行结果通过 `GET /api/v1/state` 的 `events_log` 获取。

### HTTP API 端点

#### 核心 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/health` | GET | 健康检查 |
| `/api/v1/state` | GET | 获取当前世界状态 |
| `/api/v1/tick` | GET | 获取 Tick 状态 |
| `/api/v1/intent` | POST | 提交意图 |
| `/api/v1/validate` | POST | 验证动作 |

#### 角色 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/character` | GET | 获取角色信息 |

| `/api/v1/character/register` | POST | 注册新角色 |
| `/api/v1/character/experiences` | GET | 获取经验日志（分页） |
| `/api/v1/character/dream` | GET/POST | 梦境状态/注入 |

#### 认知 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/context` | GET | 获取叙事上下文 |
| `/api/v1/attributes` | GET | 获取属性值（禁止存入记忆） |
| `/api/v1/memory/recent` | GET | 获取近期记忆 |
| `/api/v1/memory/search` | POST | 搜索记忆 |

| `/api/v1/relationship/list` | GET | 获取关系列表 |
| `/api/v1/relationship/{id}` | GET | 获取特定关系 |

#### 审核系统

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/review/pending` | GET | 获取待审核意图 |
| `/api/v1/review/{intent_id}` | POST | 提交审核结果 |
| `/api/v1/review/{intent_id}/status` | GET | 获取审核状态 |

#### 配置管理

| 端点 | 方法 | 说明 |
|------|------|------|

## Agent → Server (WebSocket)

Agent 作为客户端连接到 Server：
```
ws://cyber-jianghu-server:23333/ws?device_id={uuid}&token={auth_token}
```

## Agent → OpenClaw (WebSocket Server)

Agent 提供 WebSocket 服务端：
```
ws://localhost:23340/ws
```

### 下行消息 (DownstreamMessage)

```json
// Tick 通知
{
  "Tick": {
    "tick_id": 123,
    "deadline_ms": 50000,
    "state": { ... },
    "context": "叙事化上下文..."
  }
}

// Tick 截止通知
{
  "TickClosed": {
    "tick_id": 123,
    "received": true
  }
}
```

### 上行消息 (UpstreamMessage)

```json
// 提交意图
{
  "Intent": {
    "tick_id": 123,
    "action_type": "idle",
    "action_data": {},
    "priority": 1
  }
}
```

## 决策模式

### HTTP 模式（推荐）

OpenClaw 通过轮询获取状态并提交意图：

1. 轮询 `GET /api/v1/state` 获取世界状态
2. 基于状态生成决策
3. 提交意图 `POST /api/v1/intent`
4. 可选：使用 `/api/v1/context` 获取叙事化上下文

### WebSocket 模式（遗留）

Agent 推送 Tick 到 OpenClaw，OpenClaw 实时响应：

1. Agent 连接到 Server WebSocket
2. 接收 Tick 消息并转发到 OpenClaw
3. OpenClaw 通过 Agent WebSocket Server 提交意图
4. Agent 将意图转发到 Server
