# 玩家控制台 (Agent Control Panel)

**级别**: P2 体验增强
**模块**: `crates/agent`

## 1. 设计目标
为人类玩家或调试人员提供一个深度观察并干预 AI 角色认知的内部控制台（区别于 Server 的全局 Admin 面板，此面板聚焦于单个 Agent 内部）。

## 2. 核心机制
### 2.1 实时数据流 (SSE)
- 暴露 HTTP 端点，使用 Server-Sent Events (SSE) 协议，向浏览器前端实时推流。
- 展示内容：Agent 当前的世界观快照、HP/体力条、工作记忆列表、天魂拦截日志、以及完整的推演记录 (`SoulCycleReport`)。

### 2.2 辅助生成工具
- 辅助创建角色：输入一句简短描述（如“一个落魄的剑客”），一键调用大模型生成符合架构的完整 YAML 配置（包含世界树、初始属性雷达图、性格标签）。

### 2.3 托梦接口 (Dream)
- 暴露一个特殊的 POST 接口。
- 允许人类以上帝视角向指定 Agent 强制注入一段文本思想（如：“【系统提示】你突然觉得去京城是个好主意”）。
- 该思想会无视正常的感知来源，直接写入 Agent 的记忆深处，从而诱导其下一步行动，是测试和干预剧情走向的利器。

## 3. 架构约束
- 控制台的干预必须通过标准 API 和记忆注入进行，不能直接破坏 Agent 的内存隔离或篡改物理状态。

## 4. 代码入口
- API 路由: `crates/agent/src/infra/api/handlers.rs`
- 托梦注入: `crates/agent/src/core/lifecycle.rs` (读取 Dream 队列)
