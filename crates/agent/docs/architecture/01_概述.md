# Cyber-Jianghu Agent 概述

## 定位

Cyber-Jianghu Agent 是游戏角色的**躯体与心智的载体**，负责与 Server（天道）通信、接收世界状态、提交意图。支持两种运行模式：

- **Cognitive 模式（默认）**：内置 LLM，Agent 自主决策
- **Claw 模式**：等待外部调度器（OpenClaw）通过 WebSocket 提交 Intent

## 核心设计原则

### COI (Composition Over Inheritance)

Agent 采用组合优于继承的设计，所有功能通过组合实现：

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent (躯体 + 心智)                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌─────────────────────────────────────────────────────┐   │
│   │         AgentBuilder (统一接口)                        │   │
│   │                                                      │   │
│   │   .with_intent_translator() ← 天魂 (IntentTranslator) │   │
│   │   .with_validator()         ← 地魂 (ReflectorSoul)    │   │
│   │   .with_memory_manager()    ← Memory System           │   │
│   │   .with_reconnect_rx()      ← Claw 模式热切换          │   │
│   └─────────────────────────────────────────────────────┘   │
│                          │                                 │
│                          ▼                                 │
│   ┌─────────────────────────────────────────────────────┐   │
│   │              决策模式 (可组合的大脑)                   │   │
│   │                                                      │   │
│   │   Cognitive 模式 (内置 LLM)  │  Claw 模式 (外置)     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 三魂架构（人魂 + 天魂 + 地魂）

Agent 内部采用**三魂架构**：

- **ActorSoul (人魂/行动之魂)**：纯叙事意图生成，不接触技术 ID
- **IntentTranslator (天魂)**：LLM 翻译叙事为格式化 Intent（精确 ID 映射）
- **ReflectorSoul (地魂/反思之魂)**：三层审查（action_type → RuleEngine → LLM）

三魂同步串联在 lifecycle 主循环中，ReflectorSoul **默认启用**。

```
ActorSoul → 叙事意图 ("吃馒头充饥")
    → IntentTranslator → 格式化 Intent (action_type=eat, item_id=mantou)
    → ReflectorSoul → 三层审查
    → Approved: send_intent()
    → Rejected: narrativize_rejection → ActorSoul 重试
```

**驳回反馈叙事化**：技术性错误（"item_id 无效"）在传回人魂前被转换为自然语言（"想做的事没做成"），人魂始终不接触技术 ID。

### 通信通道

| 通道 | 协议 | 用途 |
|------|------|------|
| Agent ↔ Server | WebSocket | 实时 Tick 推送、Intent 提交 |
| OpenClaw ↔ Agent | WebSocket | Intent 提交（Claw 模式）|
| Web 面板 ↔ Agent | HTTP | 数据查询、管理操作 |

### WebSocket 实时推送（硬性要求）

- Agent 通过 WebSocket 连接 Server，接收 WorldState 推送
- Tick 无情向前，跟不上的会被抛弃
- Intent 提交必须包含**当前 tick_id**，过期会被 Server 拒绝

### HTTP API（辅助功能）

- 用于数据查询（状态、记忆、关系等）
- Web 面板、调试和测试
- 审查状态查询（`/api/v1/review/*`）
- **不是** WebSocket 的备选方案
