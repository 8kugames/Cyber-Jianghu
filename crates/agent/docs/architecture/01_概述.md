# Cyber-Jianghu Agent 概述

## 定位

Cyber-Jianghu Agent 是游戏角色的"躯体"，负责与 Server（天道）通信、接收世界状态、提交意图，并为 OpenClaw（外部调度器/大脑）提供 WebSocket 和 HTTP API 接口。

## 核心设计原则

### Agent 是躯体，OpenClaw 是大脑

```
OpenClaw (LLM/大脑) <--> Agent (躯体) <--> Server (天道)
```

- Agent 不内置 LLM 调用
- 所有决策由外部调度器（OpenClaw）负责
- Agent 只提供数据通道、状态管理和认知能力

### 通信通道

| 通道 | 协议 | 用途 |
|------|------|------|
| `/ws` | WebSocket | 实时 Tick 推送、Intent 提交 |
| `/api/v1/*` | HTTP | 数据查询（状态、记忆、关系等）|

### WebSocket 实时推送（硬性要求）

- OpenClaw **必须**通过 WebSocket 连接 Agent，确保 Tick 一致性
- Agent 内部集成 WebSocket 客户端连接 Server（默认启用）
- Agent 提供 WebSocket 服务端供 OpenClaw 连接
- Tick 无情向前，跟不上的会被抛弃

### HTTP API（辅助功能）

- 用于数据查询（状态、记忆、关系等）
- Web 面板、调试和测试
- **不是** WebSocket 的备选方案

## 端口配置

- 开发环境：`23340`
- 范围：`23340-23349`（支持多实例）

环境变量：`AGENT_PORT=23340`
