# Cyber-Jianghu Agent 概述

## 定位

Cyber-Jianghu Agent 是游戏角色的**躯体与心智的载体**，负责与 Server（天道）通信、接收世界状态、提交意图。支持两种运行模式：

- **Cognitive 模式（默认）**：内置 LLM 客户端，使用 CognitiveEngine 自主决策
- **Claw 模式**：通过 OpenClawBridge 桥接外部 LLM，使用相同的 CognitiveEngine 架构

> **架构统一**：两种模式共享相同的初始化流程（CognitiveEngine、OutcomeMemory、ChaosGenerator、回调注册），仅 LLM 客户端创建不同。

## 核心设计原则

### COI (Composition Over Inheritance)

Agent 采用组合优于继承的设计，所有功能通过组合实现：

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent (躯体 + 心智)                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌─────────────────────────────────────────────────────┐   │
│   │         AgentBuilder (统一接口)                        │   │
│   │                                                      │   │
│   │   .with_validator()         ← 天魂 (ReflectorSoul)    │   │
│   │   tool calling 工具池       ← 地魂 (能力之魂)          │   │
│   │   .with_memory_manager()    ← Memory System           │   │
│   │   .with_reconnect_rx()      ← Claw 模式热切换          │   │
│   └─────────────────────────────────────────────────────┘   │
│                          │                                 │
│                          ▼                                 │
│   ┌─────────────────────────────────────────────────────┐   │
│   │           决策模式 (统一认知架构，仅 LLM 位置差异)       │   │
│   │                                                      │   │
│   │   Cognitive (内置 LLM)      │  Claw (外置 LLM)       │   │
│   │   ↓ 统一 CognitiveEngine    │  ↓ 统一 CognitiveEngine │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 三魂架构（人魂 + 地魂 + 天魂）

Agent 内部采用**三魂架构**：

- **ActorSoul (人魂/行动之魂)**：直连 WorldState，输出含精确 ID 的结构化 Intent
- **地魂 (能力之魂)**：提供 tool calling 工具池，行动落地层（嵌入 ActorSoul）
- **ReflectorSoul (天魂/守护之魂)**：三层审查（Layer1 动作审查 → Layer2 规则校验 → Layer3 意图审查）

三魂串联在 lifecycle 主循环中。ReflectorSoul **默认启用**。

### multi-Intent Pipeline

单 tick 可提交多 Intent，顺序执行，失败回滚：
- `IntentBatchConfig`: max_intents_per_tick (默认 5), max_retries (默认 3)
- `GradedValidationConfig`: Always(强制)/Adaptive(动态)/Skip(跳过) 三策略

```
ActorSoul → 结构化 Intent (action_type=eat, item_id=mantou)
    → ReflectorSoul → 三层审查
    → Approved: send_intent()
    → Rejected: narrativize_rejection → ActorSoul 重试
```

**驳回反馈叙事化**：技术性错误（"item_id 无效"）在传回人魂前被转换为自然语言（"想做的事没做成"）。

### 通信通道

| 通道 | 协议 | 用途 |
|------|------|------|
| Agent ↔ Server | WebSocket | 实时 Tick 推送、Intent 提交 |
| OpenClaw ↔ Agent | WebSocket | Intent 提交（Claw 模式）|
| Web 面板 ↔ Agent | HTTP | 数据查询、管理操作 |

### WebSocket 实时推送（硬性要求）

- Agent 通过 WebSocket 连接 Server，接收 WorldState 推送
- Tick 无情向前，跟不上的会被抛弃
- Intent 提交必须包含**当前 tick_id**，过期会被 Server 拒绝

### HTTP API（辅助功能）

- 用于数据查询（状态、记忆、关系等）
- Web 面板、调试和测试
- 审查状态查询（`/api/v1/review/*`）
- **不是** WebSocket 的备选方案
