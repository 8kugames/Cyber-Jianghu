# Protocol 消息类型

## ServerMessage

Server 发送给 Agent 的消息：

```rust
pub enum ServerMessage {
    // 注册成功（包含游戏规则）
    Registered {
        agent_id: Uuid,
        game_rules: GameRules,
        world_building_rules: Option<WorldBuildingRules>,
        is_alive: bool,
        agent_name: Option<String>,
    },
    // 世界状态快照
    WorldState { data: WorldState },
    // 通用配置更新（统一收拢 game_rules/actions/world_building_rules/skills）
    ConfigUpdate {
        config_type: String,       // "game_rules" | "actions" | "world_building_rules" | "skills"
        update_type: String,       // "full" | "incremental"
        version: String,
        content: serde_json::Value,
        updated_items: Vec<String>,
        removed_items: Vec<String>,
    },
    // 心跳响应
    Pong { timestamp: i64 },
    // 错误消息（带结构化错误码）
    Error {
        code: String,           // 错误码常量：tick_mismatch, agent_dead, etc.
        message: String,
        current_tick_id: Option<i64>,
    },
    // 对话消息（转发）
    Dialogue { message: DialogueMessage },
    // Agent 死亡通知
    AgentDied { agent_id, cause, description, location, tick_id, died_at, rebirth_delay_ticks },
    // 即时事件（speak 等立即广播的事件）
    ImmediateEvent { event_id: Uuid, event: WorldEvent, deadline_ms: u64 },
}
```

### 错误码常量

| 常量 | 值 | 说明 |
|------|-----|------|
| ERROR_CODE_TICK_MISMATCH | "tick_mismatch" | Tick 不匹配 |
| ERROR_CODE_NOT_ACCEPTING | "not_accepting" | 服务端未接受意图 |
| ERROR_CODE_AGENT_DEAD | "agent_dead" | Agent 已死亡 |
| ERROR_CODE_RATE_LIMITED | "rate_limited" | 限速 |
| ERROR_CODE_INVALID_MESSAGE | "invalid_message" | 无效消息 |
| ERROR_CODE_DIALOGUE_FAILED | "dialogue_failed" | 对话失败 |
| ERROR_CODE_ACTION_FAILED | "action_failed" | 动作执行失败 |

### 事件类型常量

| 常量 | 值 | 说明 |
|------|-----|------|
| EVENT_TYPE_WORLD_STATE | "world_state" | 世界状态 |
| EVENT_TYPE_ACTION_RESULT | "action_result" | 动作结果 |
| EVENT_TYPE_PUBLIC_MESSAGE | "public_message" | 公开消息 |
| EVENT_TYPE_DEATH_NOTIFICATION | "death_notification" | 死亡通知 |
| EVENT_TYPE_SYSTEM_NOTIFICATION | "system_notification" | 系统通知 |
| EVENT_TYPE_DIALOGUE_START | "dialogue_start" | 对话开始 |
| EVENT_TYPE_DIALOGUE_MESSAGE | "dialogue_message" | 对话消息 |

## ClientMessage

Agent 发送给 Server 的消息：

```rust
pub enum ClientMessage {
    // 意图上报（每 Tick 一次）
    Intent {
        intent_id: Uuid,
        tick_id: i64,
        thought_log: Option<String>,
        action_type: ActionType,
        action_data: Option<serde_json::Value>,
        priority: i32,
        // multi-Intent Pipeline 支持
        subsequent_intents: Option<Vec<Intent>>,  // 后续意图列表
    },
    // 心跳
    Ping { timestamp: i64 },
    // 对话消息
    Dialogue { message: DialogueMessage },
}
```

## WorldState

每个 Tick 广播的完整世界状态快照：

```rust
pub struct WorldState {
    pub tick_id: i64,
    pub self_state: AgentSelfState,
    pub location: Location,
    pub entities: Vec<Entity>,
    pub events_log: Vec<Event>,
}
```

## IntentBatch 批次配置

multi-Intent Pipeline 配置（通过 GameRules 下发）：

```rust
pub struct IntentBatchConfig {
    pub max_intents_per_tick: usize,    // 默认 5
    pub max_retries: i32,               // 默认 3
    pub pipeline_execution_enabled: bool,
    pub partial_execution_enabled: bool,
    pub llm_validation: GradedValidationConfig,
}

pub struct GradedValidationConfig {
    pub always_types: Vec<String>,           // 强制 LLM 审核
    pub adaptive_types: Vec<String>,         // 动态审核
    pub skip_types: Vec<String>,             // 跳过 LLM 审核
    pub minimum_per_tick: usize,
    pub restricted_area_keywords: Vec<String>,
    pub high_value_item_keywords: Vec<String>,
    pub adaptive_field_mapping: HashMap<String, String>,
}
```
