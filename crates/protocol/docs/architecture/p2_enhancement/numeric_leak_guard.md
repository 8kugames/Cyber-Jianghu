# 数值泄漏防护 (Numeric Leak Guard)

**级别**: P2 体验增强
**模块**: `crates/protocol`

## 1. 设计目标
阻止 LLM 在输出文本（特别是对话或大喊内容）中直接暴漏系统底层的数字参数，维护游戏的“沉浸感”与“身心分离”原则。

## 2. 核心机制
### 2.1 正则检测 (Regex Guard)
- 通过正则表达式后置扫描 LLM 吐出的意图参数（特别是 `text` 字段）。
- 拦截诸如“扣除 10 点 HP”、“我的攻击力是 50”、“前往 node_1234”等包含硬数值或 UUID 的文本。

### 2.2 Guard Prompt 自动重试
- 发现泄漏后，并不直接让动作失败。
- 认知引擎会自动拼接警告信息（如：“警告：请勿在对话中提及具体数值，请用自然语言表达”），要求 LLM 重新生成一次。
- 若重试后依然泄漏，则触发天魂的 Layer 3 拦截，生成失败的 `ExecutionResult`。

## 3. 架构约束
- 检测过程必须轻量级，避免使用昂贵的模型进行二次校验。
- 重试次数严格受限（通常设置为 1-2 次），防止陷入死循环消耗 Token。

## 4. 代码入口
- 防泄漏处理: `crates/agent/src/soul/actor/translation.rs`
- 天魂拦截: `crates/agent/src/core/reflector_ext.rs`
