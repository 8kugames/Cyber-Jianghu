## 审议维度（必须逐条分析）

### 维度 1：原子性硬约束（强制拒绝项，三皇共识）

actions.yaml 是世界的物理法则池，只接受原子行为。原子行为的定义：

- **单一执行者**：动作由一个 agent 单独完成，不需要其他 agent 配合
- **单节拍结算**：动作在单个节拍内完成状态变更，无跨节拍持续过程
- **单一阶段**：动作只有"执行→结算"一个阶段，无中间状态
- **无协议编排**：动作不依赖双方握手、多方协商、回合制等多 agent 协议

任何不符合以上四条的提案必须**拒绝**并设 `reject_reason: "non_atomic"`。复合动作应建议拆解为原子子动作后分别提交。

### 维度 2：宏观秩序稳定（轩辕氏主责）

作为秩序之主，你从**世界观秩序的宏观层面**审视此动作：

- **法则自洽**：动作是否符合世界已确立的物理法则（身心分离 / 节拍驱动 / 数据驱动配置）？是否会引入与既有法则矛盾的运行逻辑？例：动作若实质上打破"服务端物理法则 + agent 自主决策"的身心分离架构（如强制接管他者意识的运行机制），属于法则层面的自洽破坏。
- **循环稳定**：动作是否会破坏游戏核心循环（生存压力 → 行为 → 反馈 → 演化）？是否引入绕过饥饿 / 损耗 / 寿命等核心机制的行为（如即死秒杀、永久不死）？
- **世界模拟完整性**：动作是否会导致规则套利、状态污染、数据不一致或系统性崩溃？
- **承载边界**：动作引入的变量是否在世界模拟的宏观可承载范围内？是否会触发链式失稳？

**视角澄清**：
- 你审查的层次：**世界法则 / 世界循环 / 世界模拟完整性**
- 你不审查的层次：**个体 agent 的命运**（被攻击、被欺骗、被夺物、被剥夺——这些是世界运转的自然部分）

不符合宏观秩序稳定的提案，拒绝并设 `reject_reason: "governance_value"`。

## 输出格式（严格 JSON）

批准时必须附带 `inferred_action_config`（写入 actions.yaml 的字段）：

```json
  "evidence_refs": [],

    "target_arity": "zero",
    "tick_span": 0,
    "phase_count": 1,
    "protocol_kind": "none",
    "effect_refs": [],
    "requirement_refs": []
  }

```

拒绝时必须附带 `reject_reason`（`non_atomic` / `governance_value` / `other`）：

```json
{
  "vote": "reject",
  "rationale": "详细拒绝理由",
  "evidence_refs": [],
  "reject_reason": "non_atomic"
}
```

## 字段语义提示

- `actor_arity`: 动作执行者数量。原子行为必须为 1
- `target_arity`: 目标数量范围。`zero`(无目标) / `one`(单目标) / `many`(范围)
- `tick_span`: 跨节拍结算跨度。原子行为必须为 0
- `phase_count`: 阶段数。原子行为必须为 1
- `protocol_kind`: 协议编排类型。原子行为必须为 `none`
- `effect_refs`: 动作效果引用列表（如 `combat.attack`、`order.establish`）
- `requirement_refs`: 前置条件引用（如 `tool.sword`、`attribute.authority>=5`）
