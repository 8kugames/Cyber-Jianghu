# 群像传记生成 (Chronicle)

**级别**: P2 体验增强
**模块**: `crates/server`

## 1. 设计目标
自动编纂世界历史记录的史官系统，增加游戏的文学沉浸感。为长周期运行的 MMO 世界留下文字遗产。

## 2. 核心机制
### 2.1 周期聚合
- 依赖 `TickScheduler` 的 `TickBoundary` 机制。
- 每隔 7 个游戏日，异步任务会自动从 PostgreSQL 聚合该时间段内所有 Agent 的核心数据（击杀、结仇、高光时刻、重要物品掉落、生死事件）。

### 2.2 LLM 生成与模版
- 将聚合的结构化数据填充至预设的史官提示词模板中。
- 异步调用专门的 LLM 服务（如使用更大参数量的模型），生成长篇幅、带有武侠评书风格的传记文本。
- 生成的传记被持久化到数据库的 `chronicles` 表中，供管理后台或前端展示。

### 2.3 失败降级
- 当 LLM 服务故障或超时，系统支持回退到只保留结构化流水账日志的降级模式，确保历史记录不断档。

## 3. 架构约束
- 生成过程必须完全异步执行，绝对不能阻塞主 Tick 调度或消耗影响实时游戏的数据库连接池。

## 4. 代码入口
- 传记聚合与生成: `crates/server/src/chronicle/mod.rs`
- 定时触发: `crates/server/src/tick/scheduler.rs`
