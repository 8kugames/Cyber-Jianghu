# 群像传记生成 (Chronicle)

**级别**: P2 体验增强
**模块**: `crates/server`

## 1. 设计目标
自动编纂世界历史记录的史官系统，增加游戏的文学沉浸感。为长周期运行的 MMO 世界留下文字遗产，将散落的日志转化为富有武侠叙事感的长篇评书或传记。

## 2. 核心机制
### 2.1 周期聚合触发
- 依赖 `TickScheduler` 主循环的 Tick 推进机制。
- 每隔配置的 `period_ticks` 真实秒（例如 7 个游戏日对应的时间），调度器会异步启动一次传记生成任务。
- `chronicle::collector` 会从 PostgreSQL 的 `agent_action_logs`、`world_events` 以及角色状态中聚合该时间段内的关键数据（如击杀、结仇、重要对话、掉落、生死事件）。

### 2.2 LLM 叙事生成
- 聚合后的结构化流水账日志将被组织进预设的史官提示词模板中。
- `generator.rs` 调用配置的外部大语言模型（LLM）API，将流水账“润色”为带有文学色彩、评书风格的章节文本。
- 生成的文本与元数据会被持久化保存至 `chronicles` 表，并标记了时间段与季节。

### 2.3 失败重试与异步降级
- 生成过程由后台的 `tokio::spawn` 异步接管，绝不会阻塞核心的 Tick 引擎。
- 若 LLM 服务超时或出现异常，生成任务会失败，但可以在 Dashboard 的“传记管理”界面进行手动触发或重试，确保不会出现数据丢失。

## 3. 架构约束
- 传记生成涉及重度数据库聚合和长时间的 HTTP 调用，必须处于与 `IntentWorker` 或 `TickScheduler` 完全隔离的异步任务中执行。
- 生成的传记文本仅供阅读展示，不能影响到物理世界的客观状态。

## 4. 代码入口
- 传记聚合与生成引擎: `crates/server/src/chronicle/`
- 触发机制: `crates/server/src/tick/scheduler.rs`
