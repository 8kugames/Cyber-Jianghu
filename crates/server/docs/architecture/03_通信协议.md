# Server 通信协议

## Agent → Server (WebSocket)

```
ws://server:23333/ws?device_id={uuid}&token={auth_token}
```

### 消息格式 (ClientMessage)

实际消息格式使用 `serde` 的 `#[serde(tag = "type")]` 标注，序列化为带 `type` 字段的扁平结构：

```json
// 提交意图
{
  "type": "intent",
  "intent_id": null,
  "tick_id": 123,
  "agent_id": null,
  "thought_log": "思考内容",
  "action_type": "idle",
  "action_data": null,
  "priority": 1
}

// 对话消息
{
  "type": "dialogue",
  "message": { ... }
}
```

## Server → Agent (WebSocket)

### 消息格式 (ServerMessage)

同样使用 `#[serde(tag = "type")]` 标注：

```json
// 注册成功
{
  "type": "registered",
  "agent_id": "uuid",
  "game_rules": { ... },
  "world_building_rules": { ... }
}

// 世界状态广播
{
  "type": "world_state",
  "tick_id": 123,
  "self_state": { ... },
  "location": { ... },
  "entities": [ ... ],
  "events_log": [ ... ]
}

// 错误消息
{
  "type": "error",
  "message": "错误描述"
}

// 对话消息
{
  "type": "dialogue",
  "from": "speaker_id",
  "message": "..."
}
```

## HTTP API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/health` | GET | 健康检查 |
| `/api/v1/agent/connect` | POST | 设备连接 |
| `/api/v1/agent/register` | POST | 注册 Agent |
| `/api/v1/agent/rebirth` | POST | 删除 Agent（转生） |
| `/api/v1/agent/{id}/context` | GET | 获取 Agent 上下文 |
| `/api/v1/validate-action` | POST | 验证动作可行性 |
| `/api/dashboard/stats` | GET | 仪表板统计（需读令牌） |
| `/api/dashboard/agents` | GET | 在线 Agent 列表（需读令牌） |
| `/api/dashboard/agents/offline` | GET | 离线 Agent 列表（需读令牌） |
| `/api/dashboard/agents/dead` | GET | 死亡 Agent 列表（需读令牌） |
| `/api/dashboard/agent/{id}` | GET | Agent 详情（需读令牌） |
| `/api/dashboard/agents/cleanup` | POST | 清理离线 Agent（需写令牌） |
| `/api/config` | GET | 配置列表（需读令牌） |
| `/api/config/{filename}` | GET/PUT | 获取/更新配置文件（需相应令牌） |
| `/api/admin/reload-config` | POST | 重载配置（需写令牌） |
