# 数据驱动设计

## 配置文件

| 配置文件 | 说明 |
|----------|------|
| actions.yaml | 动作定义 |
| attributes.yaml | 属性定义 |
| items.yaml | 物品定义 |
| locations.yaml | 地点图（含 parent_id、implicit_travel_cost 分层） |
| game_rules.yaml | 核心规则（生存阈值、sanity 阈值、即时事件、隐式连接默认值） |
| time.yaml | 时间系统 |
| prompt_templates.yaml | Prompt 模板配置（actor_direct、reject_feedback、survival_warnings、sanity_warnings） |
| narrative_config.yaml | 叙事配置 |
| recipes.yaml | 配方 |
| inventory.yaml | 背包配置 |
| network.yaml | 网络配置 |
| world_building_rules.yaml | 世界构建规则 |
| display_messages.yaml | 显示消息模板 |

## 动作定义示例

```yaml
actions:
  move:
    description: "移动到指定地点"
    parameters:
      - name: destination
        type: location_id
        required: true
    effects:
      - type: location_change
        target: self
    validation:
      - attribute: stamina
        min: 10
```

## 属性定义示例

```yaml
attributes:
  primary:
    - name: strength
      base: 10
      min: 1
      max: 100
    - name: agility
      base: 10
  derived:
    - name: max_health
      formula: "base_health + strength * 2"
```

## 公式引擎

支持动态公式计算：

```rust
"base_damage + strength * 0.5 + weapon_damage"
"max_health * 0.1"
"agility > 50 ? critical_chance * 2 : critical_chance"
```

## LLM 审核配置 (game_rules.yaml)

天魂分级审核策略配置：

```yaml
llm_validation:
  # 强制 LLM 审核的 action_type（高优先级）
  always_types:
    - speak
    - shout
    - whisper
  # 动态审核的 action_type（根据 action_data 判断）
  adaptive_types:
    - steal
    - trade
    - give
    - move
  # 跳过 LLM 审核的 action_type（低风险）
  skip_types:
    - idle
    - wait
  # 每 tick 至少审核的 Intent 数量
  minimum_per_tick: 1
  # 限制区域关键词（move 审核用）
  restricted_area_keywords:
    - admin
    - vault
    - secret
  # 高价值物品关键词（trade/steal/give 审核用）
  high_value_item_keywords:
    - silver
    - gold
  # Adaptive 审核字段映射（数据驱动）
  adaptive_field_mapping:
    move: target_location
    trade: item_id
    steal: item_id
    give: item_id
```

## 即时事件配置 (game_rules.yaml)

即时事件处理配置：

```yaml
immediate_events:
  # 冲突动作列表（执行这些动作时不立即回应）
  conflict_actions:
    - move
    - travel
    - gather
    - craft
    - fight
  # 呼唤关键词列表（匹配时立即回应）
  call_keywords:
    - 喂
    - 哎
    - 这位
    - 侠客
    - 朋友
  # 最大呼唤内容长度
  max_call_content_length: 50
  # 默认回应内容
  default_response: "何事？"
  # 即时路由动作列表（不占 tick 配额）
  immediate_routing_actions:
    - speak
    - whisper
```

## Intent Batch 配置 (game_rules.yaml)

multi-Intent Pipeline 配置：

```yaml
intent_batch:
  # 每 tick 最大 Intent 数
  max_intents_per_tick: 5

  pipeline_execution_enabled: true

  partial_execution_enabled: true
```

## Prompt 模板配置 (prompt_templates.yaml)

Prompt 模板从 YAML 加载，替代硬编码。启动时校验 `required_sections`，缺失则 panic（fail-fast）。

```yaml
templates:
  actor_direct:
    required_sections: [header, world_state, task, survival_rules, narrative_limits, action_reference, output_format]
    sections:
      header: "{feedback_section}你是 {agent_name}。\n{persona}"
      survival_rules: |
        - 当饥饿或口渴描述中出现紧迫措辞时，进食/饮水是最高优先级
        - eat/drink 的 action_data 必须使用精确 item_id
    truncation:
      planning_description: 100
      content_hint: 30
      summary_window: 50
```

### 四类模板

| 模板名 | 用途 | 调用方 |
|--------|------|--------|
| `actor_direct` | 人魂主 prompt（sections + truncation） | CognitiveEngine |
| `reject_feedback` | RuleEngine reject 反馈（附可用 items/nodes） | RuleEngine |
| `survival_warnings` | 饥饿/口渴紧急信号（6 种场景） | lifecycle.rs |
| `sanity_warnings` | 理智值混乱引导（mild/critical） | lifecycle.rs |

## 地图分层 (locations.yaml + game_rules.yaml)

地点支持 `parent_id` 分层（region → map → sub_scene），隐式 parent-child 连接：

```yaml
# locations.yaml
- node_id: "longmen_kitchen"
  name: "厨房"
  type: "sub_scene"
  parent_id: "longmen_inn"
  implicit_travel_cost: 0  # 覆盖全局默认值

# game_rules.yaml
agent_state:
  location:
    default_implicit_travel_cost: 1
```

`LocationGraph::get_all_neighbors()` 合并显式边 + 隐式 parent-child，`is_connected()` 自动检查双向隐式连接。

## 生存压力注入 (prompt_templates.yaml)

hunger/thirst 低于阈值时，lifecycle.rs 向 memory_context 注入紧急信号，口渴优先于饥饿：

```yaml
survival_warnings:
  sections:
    hunger_with_items: "【紧急】你极度饥饿（当前 {value}/{max}），必须立即进食！背包可用物品：{items}。"
    thirst_nothing: "【紧急】你极度口渴（当前 {value}/{max}），附近无水源！移动到有水的地点或采集。"
```

## 理智值注入 (prompt_templates.yaml)

sanity 低于阈值时注入精神状态描述，引导 LLM 产生混乱行为：

```yaml
sanity_warnings:
  sections:
    mild: "【心神不宁】你的精神状态有些恍惚（理智 {value}/{max}），思绪开始混乱。"
    critical: "【精神崩溃】你已失去理智控制（理智 {value}/{max}）！行为可能变得疯狂。"
  truncation:
    warning_threshold: 50
    critical_threshold: 30
```

## Reject 反馈增强

RuleEngine 规则失败时，从 context 提取可用 items/nodes 附在错误消息中：

```
吃东西失败：物品ID无效。背包可用物品：[馒头, 水, 银子]。请使用 eat 命令，item_id 必须是上述之一。
移动失败：目标地点ID无效。可前往的地点：[龙门客栈, 后院, 厨房]。请使用 move 命令，target_location 必须是上述之一。
```
