## 动作定义示例

```yaml
actions:
  move:
    description: "移动到指定地点"
    parameters:
      - name: destination
        type: location_id
        required: true
    effects:
      - type: location_change
        target: self
    validation:
      - attribute: stamina
        min: 10
```

## 属性定义示例

```yaml
attributes:
  primary:
    - name: strength
      base: 10
      min: 1
      max: 100
    - name: agility
      base: 10
  derived:
    - name: max_health
      formula: "base_health + strength * 2"
```

## 公式引擎

支持动态公式计算：

```rust
"base_damage + strength * 0.5 + weapon_damage"
"max_health * 0.1"
"agility > 50 ? critical_chance * 2 : critical_chance"
```

## LLM 审核配置 (game_rules.yaml)

天魂分级审核策略配置：

```yaml
llm_validation:
  # 强制 LLM 审核的 action_type（高优先级）
  always_types:
    - speak
    - shout
    - whisper
  # 动态审核的 action_type（根据 action_data 判断）
  adaptive_types:
    - steal
    - trade
    - give
    - move
  # 跳过 LLM 审核的 action_type（低风险）
  skip_types:
    - idle
    - wait
  # 每 tick 至少审核的 Intent 数量
  minimum_per_tick: 1
  # 限制区域关键词（move 审核用）
  restricted_area_keywords:
    - admin
    - vault
    - secret
  # 高价值物品关键词（trade/steal/give 审核用）
  high_value_item_keywords:
    - silver
    - gold
  # Adaptive 审核字段映射（数据驱动）
  adaptive_field_mapping:
    move: target_location
    trade: item_id
    steal: item_id
    give: item_id
```

## 即时事件配置 (game_rules.yaml)

即时事件处理配置：

```yaml
immediate_events:
  # 冲突动作列表（执行这些动作时不立即回应）
  conflict_actions:
    - move
    - travel
    - gather
    - craft
    - fight
  # 呼唤关键词列表（匹配时立即回应）
  call_keywords:
    - 喂
    - 哎
    - 这位
    - 侠客
    - 朋友
  # 最大呼唤内容长度
  max_call_content_length: 50
  # 默认回应内容
  default_response: "何事？"
  # 即时路由动作列表（不占 tick 配额）
  immediate_routing_actions:
    - speak
    - whisper
```

## Intent Batch 配置 (game_rules.yaml)

multi-Intent Pipeline 配置：

```yaml
intent_batch:
  # 每 tick 最大 Intent 数
  max_intents_per_tick: 5
  # 三魂循环最大重试次数
  max_retries: 3
  # 是否启用 Pipeline 执行
  pipeline_execution_enabled: true
  # 是否允许部分执行
  partial_execution_enabled: true
```
