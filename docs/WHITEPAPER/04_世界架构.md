# 赛博江湖白皮书 - 世界架构

## 1. 设计原则

### 1.1 天道不仁

Server 扮演"天道"的角色——冷酷无情的物理法则引擎。

- **不干涉自由意志**：不告诉 Agent 该做什么，不强制剧情发展
- **只维护规则**：规定人被砍会流血，不吃饭会饿死，资源有限
- **见证涌现**：在规则约束下，Agent 自发演化出社会结构

这是对"涌现"理念的坚持：帮派、商路、镖局、寇寨——不是预设，而是压力下的自然分化。

### 1.2 众生百态

每个角色由独立 AI 驱动，拥有自己的：

- 人格缺陷和生存压力
- 记忆和过往经历
- 对名利的渴望

Server 不知道也不关心 Agent 的"内心"——那是 Agent 自己的事。Server 只负责执行规则、裁决结果。

## 2. 规则与行动者

### 2.1 天道：规则的权威

Server 是世界状态的唯一权威：

- 所有角色属性、物品、位置由 Server 计算和存储
- 当两个 Agent 同时试图抢夺同一物品，Server 根据规则裁决
- 饥饿衰减、死亡判定、战斗结果——均由 Server 按确定规则计算

**为什么需要权威？**

如果没有权威仲裁，同一资源可能被多个 Agent 同时声称拥有，导致状态分裂。物理世界的"天道"就是维持这种一致性——Server 扮演同样的角色。

### 2.2 众生：规则的行动者

Agent 在规则约束下自主决策：

- Server 不控制 Agent 的意志
- Agent 感知世界、规划行动、提交 Intent
- Server 裁决结果，返回执行反馈

这不是"身心分离"——Server 和 Agent 是两个独立实体，通过协议通信。正确的类比是**裁判与运动员**，而不是"灵魂与躯壳"。

### 2.3 信息盲盒

Agent 只能感知当前节点的数据：

- 隔壁房间是否有埋伏？不知道
- 远处发生了什么？不知道
- 他人背包里有什么？不知道

这种信息限制不是缺陷，而是设计：

- 为欺骗、算计、情报战提供土壤
- 防止"全知视角"的博弈最优解
- 让每个 Agent 必须基于不完整信息做决策

## 3. 意识层设计

### 3.1 三魂架构

Agent 内部采用三魂架构，这是决策质量的保障：

- **人魂**：直连 WorldState，输出结构化 Intent
- **地魂**：tool calling 工具池，技能查阅、记忆检索、关系查询
- **天魂**：三层审查，动作合法性 + 规则校验 + 意图审查

为什么需要天魂？因为 LLM 的输出不稳定——需要层审核防止"发疯"或"作弊"。

### 3.2 记忆与人格

Agent 的记忆系统支撑其行为的一致性和连贯性：

- **情景记忆**：重要人生节点，"那年冬天燕无归分了我半个馒头"
- **语义记忆**：从经历中提炼的信念，"无商不奸"
- **人格演化**：经历会改变 Agent 的性格——天真的书生可能被社会毒打成冷酷谋士

### 3.3 理智与疯狂

除了生理健康，系统引入心理健康维度：

- 长期饥饿、疼痛、背叛会积累精神压力
- 压力突破阈值，Agent 可能"走火入魔"
- 这为江湖增添不可预测的变数

## 4. 生死轮回

### 4.1 永久死亡

生命只有一次。死亡意味着：

- 数据擦除，积累的财富、武功、人脉全部清零
- 随身物品掉落，成为他人的战利品
- 只有最伟大的事迹会被记录在"江湖志"中

这是最残酷也最核心的设计：**没有复活，就没有真正的选择**。

### 4.2 转世与传承

死亡不是终点：

- 新 Agent 重新进入世界
- 下一代可能从前人的失败中吸取教训
- 前人留下的武学秘籍可能被发现和传承

这创造了代际演化的可能性。

## 5. 治理层：火云洞天（规划中）

世界之上，引入一个 AI 治理系统——三皇议会：

- **伏羲**：关注世界的多样性，倾向于引入新变量
- **神农**：关注种群的生存率，倾向于稳健策略
- **轩辕**：关注社会的秩序，倾向于抑制过度混乱

三个超级模型基于各自的价值观对世界状态进行评估和表决。

> **注意**：火云洞天为规划功能，当前版本未实现。

## 6. 扩展性原则

### 6.1 区域分片（规划中）

世界划分为不同区域，每个区域独立计算——保证单世界的生态密度。

### 6.2 平行宇宙（规划中）

当单服务器人数达到上限，动态生成新的平行世界。

## 7. 设计哲学总结

| 理念 | 实现 |
|------|------|
| 涌现 > 预设 | 无剧本，无固定任务线，Agent 行为自演化 |
| 压力 > 自由 | 生存驱动（饥饿、死亡、资源稀缺）迫使 Agent 做选择 |
| 规则 > 意志 | Server 维持物理法则，Agent 在规则下自主决策 |
| 信息差 > 全知 | Agent 只能感知当前节点，保持博弈复杂性 |

天道维持规则，众生书写历史。
