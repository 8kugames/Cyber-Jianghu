# OpenClaw Cyber-Jianghu MVP 产品需求文档（PRD）

## 1. 项目愿景

**Cyber-Jianghu（虚境：江湖）**旨在构建由 AI Agent 驱动的自演化武侠社会。  
本次 MVP（Minimum Viable Product）聚焦验证**“LLM 驱动的自主 Agent 在无剧本环境下的涌现行为”**。

实验场景为**“龙门客栈”**封闭沙盒，投放 5 个异质 Agent，观察其在资源与冲突压力下是否自然产生交易、结盟、背叛与战斗。

## 2. 核心假设

MVP 旨在验证以下三个核心假设：

1. **H1（自主决策）**：LLM（通过 OpenClaw）能够理解游戏规则（`WorldState`），并基于人设生成合理的行动意图（`Intent`）。
2. **H2（行为涌现）**：在没有预设剧本的情况下，多个 Agent 之间会产生非预期社交互动（如为了食物交易、为了恩怨决斗）。
3. **H3（技术可行性）**：分布式架构（OpenClaw 客户端 + 游戏服务端）能够稳定支撑实时 MMO 体验（60 秒 / Tick）。

## 3. 场景定义：龙门客栈

### 3.1 环境设定

#### 3.1.1 地点

- **核心区域**：龙门客栈（包含大堂、后院、厨房等子区域）
- **周边区域**
  - **荒漠**：产出李广杏与木材，环境恶劣（每 Tick 扣除 HP）
  - **酒泉**：远端产粮区，产出小麦（需长途跋涉）

#### 3.1.2 环境压力

- **客栈内**：无环境伤害
- **客栈外**：荒漠风沙，每 Tick 扣除 HP（模拟恶劣天气）

#### 3.1.3 资源与生产

- **采集物**：水（后院）、木材（后院 / 荒漠）、李广杏（荒漠）、小麦（**酒泉**）
- **中间品**：面粉（小麦加工）、生面团（面粉 + 水）
- **成品**：馒头（生面团加工）、李广杏干（李广杏风干）、木剑（木材加工）
- **货币**：银两

角色模板来源于 [personas](../../crates/agent/templates/personas) 目录。摘要如下：

| 角色 | 角色定位 | 主要动机 | 典型行为倾向 |
|------|----------|----------|--------------|
| 柳云娘 | 客栈经营者 | 保利润、保秩序、积累情报 | 促成交易、压制冲突、保护资产 |
| 燕无归 | 高武力低资源刀客 | 生存与复仇并行 | 以武力或劳作换补给，极端饥饿时策略偏转 |
| 方子清 | 低武力知识型书生 | 筹盘缠、避冲突 | 以服务换食宿，借贷与求助 |
| 小翠 | 隐藏身份潜逃者 | 隐匿、避追杀、攒资源 | 伪装协作、侦察信息、风险触发后强对抗 |
| 钱三通 | 资源流通商人 | 套利、囤货、降运输风险 | 高频议价、机会收购、依附强者 |

## 4. 功能需求

### 4.1 服务端（The World）

服务端是客观物理法则的执行者，负责维护“天道”。

#### 4.1.1 Tick 引擎

- 每 60 秒推演一次世界状态

#### 4.1.2 状态管理

- **统一属性系统**：采用数据驱动属性架构（Primary / Status / Derived）
- **生理属性**：HP（生命）、Stamina（体力）、Hunger（饥饿）、Thirst（口渴）
- **物品系统**：背包管理，物品消耗与转移

#### 4.1.3 动作结算（已实现 12 种基础动作）

- `idle`：发呆（恢复体力）
- `speak`：说话（广播给同场景 Agent）
- `move`：移动（多节点跳转）
- `use`：使用物品（如吃馒头恢复饥饿、喝水恢复口渴）
- `attack`：攻击目标（扣除 HP）
- `trade`：物品 / 货币交换
- `give`：给予物品（单向转移）
- `steal`：偷窃（概率获取目标物品）
- `drop`：丢弃物品（丢弃至场景地面）
- `pickup`：拾取地面物品
- `gather`：采集场景资源（受天时 / 季节影响）
- `craft`：生产合成（消耗材料产出新物品）

#### 4.1.4 环境机制

- **生理衰减**：受季节温度影响（如冬季消耗加剧），每 Tick 动态衰减饥饿 / 口渴，恢复体力
- **环境伤害**：若当前位置环境伤害 > 0，则每 Tick 扣除 HP（可配置，默认 -5）
- **生死判定**：HP <= 0 判定死亡，清空背包并掉落全部物品至地面
- **天时系统**：动态昼夜与四季更替，系统按时辰向全局 Agent 广播时间信息（影响生理与采集产量）

#### 4.1.5 观测系统

- **Dashboard API**：提供实时在线人数、DAU、服务器运行时间、游戏内时间等监控数据

### 4.2 客户端（The Agent）

客户端是 Agent 的“肉身”，连接 OpenClaw 大脑与游戏世界。

#### 4.2.1 感知（Perception）

- 接收服务端下发的 `WorldState`

#### 4.2.2 记忆系统（Memory）

- **Working Memory**：短期工作记忆
- **Episodic Memory**：情节记忆（长期）
- **Semantic Memory**：语义记忆（已实现向量检索）

#### 4.2.3 技能系统（Skill）

- 支持动态技能生成与执行

#### 4.2.4 思考（Cognition）

- 将 `WorldState` 转化为 Prompt
- 结合 Agent 人设和记忆
- 调用 LLM（via OpenClaw）生成决策

#### 4.2.5 行动（Action）

- 将决策转化为标准 JSON 格式 `Intent` 并发送给服务端

## 5. 交互流程

1. 服务端广播 `Tick N` 的 `WorldState`（`deadline_ms` 为关单绝对时间戳）
2. 客户端接收状态，调用 LLM 推理
3. 客户端在 `deadline_ms` 前提交 `Intent`（`tick_id` 必须匹配当前 `accepting_tick_id`）
4. 收集窗口关闭，服务端结算全部 `Intent`
5. 服务端持久化状态，结算事件在 `Tick N+1` 的广播中推送

## 6. 成功标准

- [ ] 系统稳定运行 24 小时且无崩溃（通过 Dashboard 监控验证）
- [ ] Agent 能够稳定存活（具备吃喝决策能力）
- [ ] 观察到至少一次**非预设复杂交互**

### 6.1 验收口径（量化）

#### 6.1.1 运行稳定性

- 连续运行时长：`>= 24 小时`
- 崩溃次数：`= 0`
- Tick 完成率：`>= 99%`（应执行 Tick / 实际执行 Tick）
- 意图超时率：`<= 5%`（30 秒窗口内未返回 `Intent` 的占比）

#### 6.1.2 生存能力

- 观测窗口：连续 `>= 240 Tick`（约 4 小时）
- 存活下限：任意时刻存活 Agent 数 `>= 3 / 5`
- 自主补给证据：每个存活 Agent 在窗口内至少完成 `>= 3` 次 `use`（食物或水）

#### 6.1.3 复杂交互（Emergent Interaction）

- 复杂交互定义：一次事件链中至少包含 `>= 3` 个原子动作，且涉及 `>= 2` 个 Agent，动作类型覆盖交易/冲突/协作中的 `>= 2` 类
- 最低通过阈值：在 `>= 240 Tick` 观测窗口内记录到 `>= 1` 次符合定义的事件链

## 7. 未来规划（Roadmap）

### 7.1 Phase 1：龙门客栈（Current MVP）

验证核心：生存本能与微观互动

- [x] **基础架构**：Tick 引擎、WebSocket 通信、PostgreSQL 状态持久化
- [x] **生理系统**：饥饿、口渴、体力衰减与恢复
- [x] **动作系统**：12 种基础动作（Idle、Speak、Move、Give、Steal、Use、Drop、Pickup、Gather、Craft、Attack、Trade）
- [x] **环境互动**：环境伤害机制、地面物品系统（丢弃 / 死亡掉落）
- [x] **世界扩张**：多节点地图支持（客栈、荒漠、**酒泉**）
- [x] **天时法则**：昼夜与四季更替（影响生理衰减与采集产出）
- [x] **基本经济**：完整产销链条（采集—加工—消耗），物品掉落与自然腐烂
- [x] **Agent 基础**：感知—思考—行动循环，短期 / 长期记忆

#### 7.1.1 非目标（Out of Scope）

以下内容明确不纳入当前 MVP 验收范围：

- **宏观系统**：不验证稳定价格体系、税收制度、宏观政治治理
- **生命周期闭环**：不验证永久死亡、传承、多代际资产转移
- **超大规模架构**：不验证区域分片、动态扩容、海量并发容量
- **UGC 与高阶心智**：不验证第三方 Skill 生态治理与长期人格演化

### 7.2 Phase 2：清明上河图（Alpha）

验证核心：经济循环与分工

- **世界扩张**
  - **感知迷雾**：仅能感知当前节点信息，信息传播依赖 Agent 交流
- **经济系统（The Sink）**
  - **三级产业**：服务业（镖局 / 情报）
  - **动态物价**：供需决定价格，无官方定价
  - **物品损耗**：装备磨损
- **记忆与社交**
  - **关系网络**：量化好感度（-100 \~ 100），动态标签（至交 / 死敌）
  - **语义记忆**：向量检索与知识库
- **生命周期**
  - **年龄系统**：引入生老病死，老年阶段生理机能衰退

### 7.3 Phase 3：江湖大世（Beta）

验证核心：宏观社会结构与政治

- **武学体系**
  - **内功 / 招式 / 轻功**：策略性战斗，非单纯数值比拼
  - **永久伤残**：重伤导致属性永久削弱，增加战斗风险成本
- **社会组织**
  - **帮派与门派**：建立组织，控制领地，征收税赋
  - **宏观调控**：“自然灾害”（如地震、洪水）触发社会动荡，影响经济与社会结构
- **生死轮回**
  - **永久死亡**：死亡即销号，资产清空或掉落
  - **传承系统**：“江湖志”记录传奇事迹

### 7.4 Phase 4：混沌纪元（Live）

验证核心：无限扩展与数字永生

- **技术架构**
  - **区域分片**：动态负载均衡，支持海量 Agent 在线
  - **动态生成**：动态生成新世界以容纳更多人口
- **UGC 生态**
  - **标准化接入**：社区通过 Skill 协议接入异构 LLM 或自定义逻辑
  - **内容共创**：玩家提交新武学、物品定义
- **高阶心智**
  - **反思**：周期性复盘，动态更新 System Prompt，实现性格演化
  - **理智**：引入心理健康值，过低可能触发“走火入魔”（随机疯狂行为）
- **治理架构（火云洞天）**
  - **AI 三皇议会**：引入三个超级模型分别扮演“伏羲”（演化）、“神农”（生存）、“轩辕”（秩序）
  - **自动表决机制**：针对“世界观偏移”“涌现事件传播”“宏观调控”发起投票（2 / 3 通过）
  - **小千世界关联**：探索与 [Cyber-Worlds-Microcosm](https://github.com/8kugames/Cyber-Worlds-Microcosm) 的互联互通
